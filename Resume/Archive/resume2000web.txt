Project Experience

- WRM-CA, War Reserve Materiel Capability Assessment (1999-2000) 

System architect and lead developer for this distributed component system. The primary client interface is pure Java using Swing components.  Portions of the server components are also accessible through a CORBA interface to provide a link with UTC-DT and other non-Java clients.

CORBA and Java developer for this distributed component system.  A CORBA interface was added to an existing Win32 application (Delphi) to use it as a server component in this system built on a Java/CORBA architecture. 

- UTC-DT, Unit Type Code Development and Tailoring (1996-1999)

System architect and developer of this peer to peer system. UTC-DT automates the initial generation of equipment requirements based on user defined rules and enables individuals to tailor those requirements.  It uses a custom peer-to-peer socket communication protocol developed for this project to allow multiple users across a LAN or the Internet to view and update the same detailed information. After the initial data load, only the changes to the data are sent around the network to reduce the bandwidth requirements.
