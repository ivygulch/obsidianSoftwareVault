# Stillwater Mining - MetalsQMS 2006-2012

## Context

Six-year engagement as independent Java architect and lead developer for MetalsQMS — a quality management system supporting the recycling line of business at Stillwater Mining (platinum and palladium producer). Contracted through prime CDSMT with a direct working relationship to Stillwater's operations team. Integrated with Stillwater's JD Edwards ERP for shipment intake, recoverable-metal assay tracking, and settlement.

## Role

Primary architect and lead developer for MetalsQMS — system design, Java implementation, mixed-database backend (IBM DB2 production, Apache Derby local/test), JD Edwards ERP integration, UAT support, and long-term maintenance across evolving Stillwater operations.

## Achievements

- Designed and built MetalsQMS, a Java-based quality management system covering Stillwater Mining's recycling business line. #architecture #java #enterprise
- Integrated MetalsQMS with Stillwater's JD Edwards ERP for shipment intake, recoverable-metal assay tracking, and settlement workflows. #erp-integration #jd-edwards #enterprise-integration
- Engineered the system on a mixed database backend (IBM DB2 for production, Apache Derby for local and test) with an open-source plus proprietary component stack. #database #java
- Maintained and extended the system over six years across multiple UAT cycles, bug investigations, and operational workflow changes. #maintenance #long-term-client #uat

## Tech Stack

Java, IBM DB2, Apache Derby, Glassfish, JBoss, JD Edwards ERP, SQL, open-source plus proprietary Java components.
