# Pillar - Agentic Tooling

## Narrative

### Agentic Tooling Across the App Lifecycle

- **Walmart internal evaluation and rollout, late 2024 – February 2026.** Personal experimentation began with Microsoft Copilot and Visual Studio, alongside evaluation of various Xcode plugins (found deficient for iOS-specific workflows). During 2025, led deep evaluation, training, and integration of Builder.io — connecting Walmart's design library with Figma to generate and update iOS views. Engineering management mandated Builder.io for all new iOS view work and as an option for updating existing views; adoption was actively growing across the 120+ iOS engineer organization at time of departure. Trained 80+ engineers directly through live sessions, with additional reach via session recordings. Participated in engineering working groups evaluating tools for broader adoption.
- **iOS incident management POC (volunteer fire departments), 2026–present.** Full lifecycle, solo. Built almost exclusively via GitHub Speckit for specification generation and Claude Code for implementation, in SwiftUI + Supabase. Delivered working prototype from initial idea in 30 days — roughly a 5x improvement in time-to-market versus traditional solo development. The value I bring is requirements-to-spec translation and architectural judgment; the tools are the implementation partner.
- **CLEAR senior iOS subcontract, 2026–present.** Use Claude Code and Codex to research the flagship iOS codebase, develop architectural proposals and PRDs, and apply to day-to-day code assistance and test development. Mentor the 6-engineer iOS team on bringing agentic tooling into day-to-day work.
- Apply agentic tools across planning, specification, implementation, testing, verification, and deployment — as accelerators for engineering judgment, not substitutes for it.
